## Dashboard Purpose

### What This Is

This is an **admin/ops monitoring dashboard** for a **fully automated** newsletter pipeline. The AI Editor runs automatically on schedule - this dashboard exists to:

1. **Monitor** - See that automation is running correctly
2. **Debug** - View detailed logs when something fails
3. **Intervene** - Retry failed jobs, override selections when needed
4. **Configure** - Edit system prompts for the AI models
5. **Audit** - Review historical executions and data

### What This Is NOT

- Not an editorial tool for manually creating newsletters
- Not a consumer-facing product
- Not replacing the automation - just monitoring it

### Pain Points This Solves (from n8n)

- Can't see pipeline status at a glance
- Multiple workflows to check separately
- No centralized logging
- Hard to debug failures
- No preview before send
- Can't edit prompts without code changes

## Design Goals

### Primary Objectives

1. **Visibility** - At a glance, understand pipeline status and issues
2. **Logging** - Detailed logs with error codes for every execution
3. **Control** - Ability to intervene, retry, edit prompts at any step
4. **Data Access** - Mirror Airtable data directly in dashboard

### Design Principles

- **Monitoring-first** - Everything automated, dashboard is read-mostly
- **5-step organization** - Navigation mirrors the pipeline steps
- **Detailed logging** - Every execution logged with structured error codes
- **Prompt editing** - Direct control over AI behavior from UI
- **Airtable mirroring** - See the actual data without leaving dashboard

## Core Features

### 1. Pipeline Overview Dashboard

At-a-glance view of:
- Current pipeline status (all 5 steps)
- Today's issue subject line and slot headlines
- Upcoming scheduled executions
- Recent activity/logs
- Any errors requiring attention

### 2. Step-by-Step Monitoring

Each step page shows:
- Execution history (last 30 days)
- Detailed logs with timestamps
- Error codes and stack traces
- Retry controls
- **System prompt editor**

### 3. System Prompt Management (CRITICAL)

Ability to edit AI prompts directly from UI:
- Pre-filter prompt (Gemini)
- Slot 1-5 agent prompts (Claude)
- Content cleaning prompt (Gemini)
- Decoration prompts (Claude)
- Image generation prompts

**Changes saved immediately apply to next execution.**

### 4. Airtable Data Mirroring

Embedded views of key Airtable tables:
- Newsletter Stories (with slot eligibility)
- Selected Slots (today's selections)
- Decoration table (AI-generated content)
- Source Credibility scores
- Newsletter Issues (history)

### 5. Detailed Logging

Every execution logged with:
- Timestamp
- Step/job name
- Status (success/failure/warning)
- Error code (if failed)
- Duration
- Input/output data
- Full stack trace (on error)

## System Prompt Management

### How It Works

1. **Prompts stored in PostgreSQL** - Not Airtable, for version control
2. **Version history** - Every change tracked with timestamp and user
3. **Rollback capability** - Revert to any previous version
4. **Immediate effect** - Next execution uses new prompt

### Prompt Storage Schema

```sql
CREATE TABLE system_prompts (
  id UUID PRIMARY KEY,
  step_number INT NOT NULL,       -- 1, 2, 3, 4, 5
  slot_number INT,                -- NULL for step-level, 1-5 for slot-specific
  prompt_name VARCHAR(100),       -- e.g., "slot_1_prefilter", "decoration_headline"
  prompt_text TEXT NOT NULL,
  model VARCHAR(100),             -- e.g., "gemini-3-flash-preview"
  temperature DECIMAL(3,2),
  created_at TIMESTAMP DEFAULT NOW(),
  created_by VARCHAR(255),
  is_active BOOLEAN DEFAULT true
);

CREATE TABLE prompt_history (
  id UUID PRIMARY KEY,
  prompt_id UUID REFERENCES system_prompts(id),
  prompt_text TEXT NOT NULL,
  changed_at TIMESTAMP DEFAULT NOW(),
  changed_by VARCHAR(255),
  change_reason TEXT
);
```

## Logging & Error Codes

### Log Levels

| Level | Color | Use Case |
|-------|-------|----------|
| DEBUG | Gray | Verbose debugging (hidden by default) |
| INFO | Blue | Normal operations |
| WARN | Yellow | Non-fatal issues |
| ERROR | Red | Failed operations |
| FATAL | Red+Bold | System-level failures |

### Error Code Structure

```
[STEP]_[CATEGORY]_[NUMBER]

Examples:
- PREFILTER_AIRTABLE_001  - Airtable connection failed
- PREFILTER_GEMINI_001    - Gemini API rate limit
- PREFILTER_GEMINI_002    - Gemini API invalid response
- SELECT_CLAUDE_001       - Claude API timeout
- SELECT_CLAUDE_002       - Claude returned invalid JSON
- DECOR_IMAGE_001         - Gemini image generation failed
- DECOR_IMAGE_002         - OpenAI fallback also failed
- COMPILE_TEMPLATE_001    - HTML template parsing error
- SEND_MAUTIC_001         - Mautic API authentication failed
- SEND_MAUTIC_002         - Mautic campaign creation failed
```

### Error Code Reference (stored in code + displayed in UI)

| Code | Message | Resolution |
|------|---------|------------|
| PREFILTER_AIRTABLE_001 | Airtable connection failed | Check API key, retry |
| PREFILTER_GEMINI_001 | Gemini API rate limit | Wait 60s, retry |
| SELECT_CLAUDE_001 | Claude API timeout | Retry with backoff |
| DECOR_IMAGE_001 | Gemini image failed | Auto-fallback to OpenAI |
| SEND_MAUTIC_001 | Mautic auth failed | Check OAuth tokens |

## Technical Decisions

### Resolved Questions

| Question | Decision | Rationale |
|----------|----------|-----------|
| Sidebar vs tabs | **Left sidebar** | Mirrors 5-step pipeline, always visible |
| WebSocket vs polling | **Polling (5-10s)** | Simpler, sufficient for monitoring |
| Modal vs drawer | **Drawer** | Keeps context visible |
| Mobile support | **Desktop-only V1** | Admin tool, not consumer |
| Dark mode | **Skip V1** | Unnecessary complexity |
| Keyboard shortcuts | **Skip V1** | Nice-to-have, not essential |

### Tech Stack

- **Framework:** Next.js 16 with App Router
- **Styling:** Tailwind CSS
- **Components:** shadcn/ui (Radix primitives)
- **State:** React Query for server state
- **Tables:** TanStack Table for data grids
- **Code editor:** Monaco (for prompt editing)
